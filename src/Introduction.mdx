# CMS Content Library

Welcome to the **CMS Content Library** - a powerful React + Tailwind CSS system for building dynamic landing pages from JSON configuration.

## ğŸš€ Quick Start

The CMS library allows you to create entire pages using simple JSON objects:

```tsx
import { CMSContent, defaultComponentRegistry } from '@picanteverde/cms';

const content = [
  {
    type: 'Hero',
    title: 'Welcome',
    subtitle: 'Build pages with JSON',
    call_to_action: 'Get Started'
  }
];

<CMSContent
  content={content}
  components={defaultComponentRegistry}
/>
```

## ğŸ“š Key Concepts

### Components (Uppercase)

Components are React components that start with **uppercase** letters:

- `Hero` - Banner sections
- `Container` - Layout wrapper
- `Card` - Card containers
- `Button` - Interactive buttons
- `Text`, `Title` - Typography
- And more...

### Templates (lowercase)

Templates are reusable patterns that start with **lowercase** letters:

- `hero` - Pre-configured hero sections
- `featureCard` - Feature cards with icon/title/description

Templates use placeholder syntax (`$variable`) that gets replaced with your data:

```json
{
  "type": "featureCard",
  "icon": "âš™ï¸",
  "title": "My Feature",
  "description": "Feature description"
}
```

### Nested Components

Any property can be a component instead of a simple value:

```json
{
  "type": "Hero",
  "title": {
    "type": "Text",
    "text": "Custom Title",
    "size": "4xl",
    "weight": "bold"
  }
}
```

### Content Property

The special `content` property renders as children:

```json
{
  "type": "Container",
  "content": [
    { "type": "Text", "text": "First item" },
    { "type": "Text", "text": "Second item" }
  ]
}
```
